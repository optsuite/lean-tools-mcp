# Lean Tools MCP Server — 项目总结

## 一、项目概述

本项目实现了一个面向 **Lean 4** 形式化证明的 **MCP（Model Context Protocol）服务器**，可被 Cursor、Claude Desktop 等 AI 编程助手直接调用。服务器提供 **21 个工具**，涵盖 LSP 交互、定理检索、LLM 集成、Lean 元编程四大类能力。

**核心指标：**
- Python 源码约 **4,600 行**（不含测试），Lean 元编程代码约 **1,800 行**
- 测试代码约 **2,900 行**，共 **193 个测试用例全部通过**
- 支持 **stdio** 和 **SSE/HTTP** 双传输模式
- 仓库地址：https://github.com/TropicalFatFish/lean-tools-mcp （私有）

## 二、整体架构

```
┌──────────────────┐         ┌──────────────────────────────────────────┐
│    MCP 客户端     │◄──────►│           MCP Server（Python）            │
│  Cursor / Claude  │ stdio  │                                          │
│  远程客户端       │ SSE    │  ┌────────────┐    ┌──────────────────┐  │
│                  │        │  │  工具注册表  │    │   LSP 连接池      │  │
└──────────────────┘         │  │  21 个工具  │    │  lean --server ×N │  │
                            │  └────────────┘    │  （轮询调度）       │  │
                            │                    └──────────────────┘  │
                            │  ┌────────────┐    ┌──────────────────┐  │
                            │  │ 外部搜索    │    │   LLM 客户端      │  │
                            │  │ HTTP 客户端 │    │ 多提供商+密钥轮换  │  │
                            │  │ + 滑动窗口  │    │ + 重试+降级       │  │
                            │  │   限流器    │    └──────────────────┘  │
                            │  └────────────┘                          │
                            │  ┌──────────────────────────────────┐   │
                            │  │  Lean 元编程 CLI 工具（子进程调用） │   │
                            │  │  havelet_generator / definition_tool│  │
                            │  │  decl_exporter                     │   │
                            │  └──────────────────────────────────┘   │
                            └──────────────────────────────────────────┘
```

**设计要点：**
- **LSP 池化架构**：不使用 REPL，而是通过 `lean --server` 的 LSP 实例池来响应请求。每个 LSP 实例内存开销固定，支持大规模并行调用。
- **异步全栈**：基于 Python `asyncio`，所有 I/O 均异步，包括 LSP 通信、HTTP 搜索、LLM 调用。
- **双传输**：stdio 用于本地 IDE 集成，SSE/HTTP 用于远程部署。

## 三、分阶段实现内容

### Phase 1：基础框架与 LSP 协议层

| 模块 | 文件 | 说明 |
|------|------|------|
| JSON-RPC 2.0 协议 | `lsp/protocol.py` | 完整的消息编解码、请求-响应匹配、通知分发 |
| LSP 客户端 | `lsp/client.py` | 管理单个 `lean --server` 进程，实现 `initialize`、`textDocument/didOpen` 等协议 |
| 文件管理器 | `lsp/file_manager.py` | 管理 LSP 文档版本、临时文件、diagnostics 事件 |
| LSP 类型 | `lsp/types.py` | Position、Range、Diagnostic、PlainGoal 等数据结构 |
| 配置系统 | `config.py` | 环境变量 + config.json + CLI 参数三级配置 |

### Phase 2：LSP 连接池与核心工具

| 模块 | 文件 | 说明 |
|------|------|------|
| 连接池 | `lsp/pool.py` | 管理 N 个 LSP 实例，round-robin 调度，故障检测 |
| 证明状态 | `tools/goal.py` | `lean_goal`、`lean_term_goal` — 获取证明目标 |
| 诊断信息 | `tools/diagnostics.py` | `lean_diagnostic_messages` — 编译错误/警告 |
| 悬停信息 | `tools/hover.py` | `lean_hover_info` — 类型签名和文档 |
| 自动补全 | `tools/completions.py` | `lean_completions` — IDE 补全 |

### Phase 3：文件操作与代码执行

| 模块 | 文件 | 说明 |
|------|------|------|
| 文件操作 | `tools/file_ops.py` | `lean_file_outline`（声明骨架）、`lean_file_contents`（文件内容）、`lean_declaration_file`（符号定义跳转）、`lean_local_search`（本地声明搜索） |
| 代码执行 | `tools/run_code.py` | `lean_run_code` — 运行独立代码片段 |
| 多策略尝试 | `tools/multi_attempt.py` | `lean_multi_attempt` — 同时尝试多个证明策略，返回各自的目标状态 |

### Phase 4：外部搜索工具

| 模块 | 文件 | 说明 |
|------|------|------|
| 滑动窗口限流器 | `clients/rate_limiter.py` | 通用异步限流，防止触发 API 频率限制 |
| HTTP 搜索客户端 | `clients/search.py` | 5 个外部 API 的 HTTP 客户端封装 |
| 搜索工具 | `tools/search.py` | `lean_leansearch`（自然语言搜索 Mathlib）、`lean_loogle`（类型签名搜索）、`lean_leanfinder`（语义搜索）、`lean_state_search`（基于目标状态查找引理）、`lean_hammer_premise`（自动推理前提建议） |

### Phase 5：LLM 集成与统一搜索

| 模块 | 文件 | 说明 |
|------|------|------|
| LLM 客户端 | `llm/client.py` | 多提供商支持（DeepSeek、OpenAI 等）、密钥轮换、指数退避重试、提供商降级 |
| 统一搜索 | `tools/unified_search.py` | `lean_unified_search` — 并行调用 LeanSearch + Loogle + LeanFinder，结果去重合并 |
| LLM 工具 | `tools/llm_tools.py` | `lean_llm_query` — 向 LLM 请求 Lean 4 / 数学推理辅助 |

### Phase 6：Lean 元编程工具移植

| 模块 | 文件 | 说明 |
|------|------|------|
| Lean 项目 | `lean/` 目录 | 独立 Lake 项目，包含 4 个 Lean 库和 3 个可执行文件 |
| HaveletGenerator | `lean/src/HaveletGenerator/` | 提取 `have`/`let` 绑定为顶层声明（基于 InfoTree 遍历） |
| DefinitionTool | `lean/src/DefinitionTool/` | 分析定理依赖关系（递归解析 `Expr` 中的常量引用） |
| DeclExporter | `lean/src/DeclExporter/` | 批量导出 Lean/Mathlib 模块声明为 JSONL |
| StateExpr | `lean/src/StateExpr/` | 自定义策略，序列化完整的证明状态表达式树 |
| Python 封装 | `tools/lean_meta.py` | `lean_havelet_extract`、`lean_analyze_deps`、`lean_export_decls` — 子进程调用 Lean CLI + LEAN_PATH 动态构建 |

### Phase 7：SSE 传输、配置完善、文档

| 模块 | 文件 | 说明 |
|------|------|------|
| SSE 传输 | `server.py` | 基于 Starlette + uvicorn，提供 `/sse`（事件流）、`/messages`（JSON-RPC）、`/health`（健康检查） |
| 配置管理 | `config.py` | 修复 `LEAN_PATH` 环境变量冲突，新增 SSE host/port 配置，完善 CLI 参数 |
| 项目文档 | `README.md` | 完整的使用指南、工具参考、架构图、配置说明 |
